
# GCPC — фикс детектора (две руки), жесты по ТЗ, OSD

## Что исправлено vs предыдущая версия
1. **Добавлен детектор (MediaPipe Hands)** → лэндмарки **привязаны к руке**, прогнозы **обнуляются**, когда руки нет.
2. **Две руки**: возвращается до 2 рук с `label`/`score`. Разделение Left/Right с бэкап‑эвристикой.
3. **Дальность**: у MediaPipe палм‑детектор масштаб‑инвариантный; на практике детектит дальше, чем регрессионные чисто‑ОННХ.
4. **Стабилизация**: гейтинг по уверенности; при отсутствии рук — пустые результаты (никакой «летучки» точек).
5. **Палм‑детектор** фактически «вернулся», т.к. в MediaPipe это отдельный этап перед лэндмарками.

## Установка
```bash
pip install -r requirements.txt
```
(Для MediaPipe на Windows нужен пакет `mediapipe`).

## Модель
* По умолчанию используется **MediaPipe Hands** (модели идут внутри пакета). CUDA‑эксперименты с оннх‑лэндмарками оставлены в `tracker_onnx.py`, но требуют **отдельного детектора**.

## CUDA / ONNXRuntime
* Для GPU-версии установи `onnxruntime-gpu` и драйверы CUDA. При запуске onnx-моделей в консоль теперь выводится активный провайдер (`[ONNX] Активен CUDAExecutionProvider ...`).
* Если CUDA недоступна, сообщение с причинами появится прямо в консоли и произойдёт явный фолбек на CPU. Чтобы сделать CUDA обязательной, можно выставить переменную окружения `GCPC_REQUIRE_CUDA=1` и убедиться, что перед запуском `CUDAExecutionProvider` присутствует в `python -c "import onnxruntime as ort; print(ort.get_available_providers())"`.

## Запуск
```bash
python -m app.main
```

## Управление (бимануально)
1. **Старт** — обе ладони (две `OPEN_PALM`).
2. **Набор** — правая показывает жест → в OSD `CAND:`.
3. **Подтверждение** — левый `PINCH_TAP` → шаг добавляется в `BUF`.
4. **Отмена одного** — слева `OPEN_PALM → FIST` в ~0.9с.
5. **Отправка** — два кулака одновременно.
